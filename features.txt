                    DETAILED FUNCTIONAL SPECIFICATIONS (PROPOSED)

A. CORE PACKAGE MANAGEMENT BEHAVIOR (USER DEFINED)
--------------------------------------------------------------------------------
* Restricted Package Manager Access:
  - Users are blocked from running raw 'apt' commands directly.
  - The custom Axiom package manager wrapper ('axm') manages all underlying
    apt and flatpak operations to prevent system pollution and keep package tracking clean.
  - Pristine System Isolation: Userland packages are strictly prohibited from being installed to the host system. The host OS remains completely pristine; all userland packages must run inside sandboxes or containerized distropaks.
* axm pkg-rfsh:
  - Updates repository index files and metadata for all configured sources (system APT repositories and Flatpak registries).
  - Version Drift Resilience: Automatically handles Debian Sid's inevitable repository version shifts by matching available mirrors dynamically without transaction failure.
* axm pkg-updt:
  - Simulates a full system distribution upgrade (dist-upgrade).
  - Dry-runs the upgrade, audits package dependencies, and alerts the user of any conflicts or security vulnerabilities (CVEs).
  - Automatically downloads all upgrade packages to the local cache without installing them, keeping the download pre-staged.
* axm apply:
  - Resolves package differences and installs all downloaded/pre-staged updates from the local cache.
  - Compilation Prompts: If compilation tasks (such as Gentoo source builds) are detected during state synchronization, axm prompts the user on how they want to proceed (e.g., compile immediately, compile in the background, or defer the build).
* axm pkg-add:
  - Adds a new package to the tracking list after performing dependency and CVE checks.
  - Interactive Source Selector: Shows options 1-6 for containers/distropaks (latest, sandbox, stable, secure, rolling, fast, compile) and a protected option [(ctrl + ;)!system!] for native system installs.
  - Automatic Bypass: If the package is only available on a single container/sandbox, it bypasses the source selection menu and selects it. If only available on "system", it goes straight to the system warning screen.
  - System Installation Guard: If the "system" layer is targeted, requires pressing the physical keyguard "ctrl + ;" (requires physical reach) and typing the confirmation string "<3 Proceed" (Deltarune-inspired) to bypass the warning.
  - Fuzzy Matching Resolution: If the exact package name is not found on a chosen distropak, warns the user of the closest match and requests confirmation. If rejected (pressing "N"), prompts with:
    * (E)nter correct package name
    * (L)ist other matching results
    * (R)eturn to warning
    * (C)ancel pkg-add
* axm/packages.d Directory Scope Constraint:
  - '/etc/axiom/axm/packages.d/' is strictly reserved for tracking packages and flatpaks only. No auxiliary config variables or parameters are allowed.

B. EPHEMERAL SANDBOX SHELLS (axm run)
--------------------------------------------------------------------------------
* Description: Run untrusted CLI commands or compile scripts inside a temporary,
  throwaway container.
* Mechanics:
  - 'axm run <image_name>' instantly pulls/runs a Podman container, mounts
    the current directory host-path, and launches an interactive shell.
  - The container's overlay disk storage is automatically purged upon shell exit,
    leaving no trace on the host machine.

C. STATE PORTABILITY BUNDLER (axm export --iso)
--------------------------------------------------------------------------------
* Description: Package the complete system configuration, themes, wallpaper,
  and package manifest into a custom bootable Live ISO.
* Mechanics:
  - The tool reads files under 'axm/packages.d/', replicates desktop customizations into '/etc/skel',
    and wraps the Cubic or 'live-wrapper' build flow.
  - Output is a standard bootable ISO matching the user's exact system state
    for easy deployment to other machines.

D. BOOT PERFORMANCE DIAGNOSTIC CONSOLE (axm boot-profile)
--------------------------------------------------------------------------------
* Description: Benchmark, profile, and diagnose system boot delays.
* Mechanics:
  - An interactive terminal dashboard wrapping 'systemd-analyze blame' and
    'systemd-analyze critical-chain'.
  - Identifies bottlenecks (slow service starts, network handshakes) and suggests
    systemd profile adjustments to optimize startup times.

E. DISTROPAK SOURCES AND LABELS
--------------------------------------------------------------------------------
* Source Categories & Description Labels (installdesc):
  - Axiom: "System (NOT RECOMMENDED FOR USERLAND PACKAGES, USE DISTROPAK INSTEAD)"
  - Universal [flatpak, snap, appimage]: "sandbox"
  - Arch/CachyOS [dependent on wizard selection]: "rolling"
  - Fedora: "secure"
  - Rawhide: "latest"
  - Debian: "stable"
  - Gentoo: "compile"
  - Alpine: "fast"

F. CROSS-DISTRO METADATA TRANSLATION DATABASE
--------------------------------------------------------------------------------
* Generation (Serverless / GitHub Actions):
  - Weekly cron job fetches official repository metadata databases directly (no web scraping).
  - Normalizes and groups packages by shared homepage URLs, binaries, and source archive names.
  - Sorts and compiles the resulting cross-references into a flat binary table ('packages.bin').
  - Hosts the compiled binary table file on GitHub Pages / CDN for free distribution.
* Lookup Mechanics (axm client):
  - 'axm pkg-rfsh' downloads the compressed 'packages.bin' file to '/var/lib/axm/packages.bin'.
  - 'axm' maps the file into memory using 'mmap()' to eliminate file parsing overhead.
  - Uses an O(log N) binary search on the sorted structs in the memory-mapped file to resolve package names in under 1ms.
  - If a package has no homepage or fails correlation, falls back to name-similarity checks or the fuzzy-matching resolution prompt.

G. AXIOM-SOS & RECOVERY CONSOLE (axiom-blackbox)
--------------------------------------------------------------------------------
* Description: A unified, hypersorted diagnostics log database collecting errors
  and warnings across all host filesystems, sandboxes, and distropak containers.
* Mechanics:
  - Axiom-SOS: A system daemon that aggregates log outputs (stdout/stderr/syslog)
    from all container layers and the host into a single, structured database. Logs
    are sorted by severity from worst to best (e.g. kernel panics at the top, chatty
    driver warnings at the bottom).
  - axiom-blackbox: A recovery script designed to run from any live system or secondary OS. 
    It parses the mounted target disk partition, extracts the Axiom-SOS indexed database stream,
    attempts to isolate the root failures, and displays a clean error dependency tree showing
    what triggered the crash sequence.

H. AXIOM-MODHUB & KERNEL SWAP (axiom-modhub)
--------------------------------------------------------------------------------
* Description: A unified hardware integration center to manage drivers, firmware,
  and kernels.
* Mechanics:
  - axiom-modhub: Installs hardware drivers, verifies compatibility, and handles
    motherboard BIOS updates.
  - Kernel Swap (axiom-modhub -kS): Presents an interactive menu of all installed
    kernels. Selecting a target kernel utilizes kexec to instantly load the new
    kernel and restart the OS in under 3 seconds without a full hardware reboot.

I. AUTO-HEALING CONFIG WATCHER (axiom-autoheal)
--------------------------------------------------------------------------------
* Description: A background service protecting system configuration and software
  state from drift.
* Mechanics:
  - axiom-autoheal: Periodically compares the host filesystem and configuration state
    against the `/etc/axiom/axm/packages.d/` blueprint.
  - If a file drifts or becomes corrupted, the daemon automatically runs a silent
    transaction to heal the file back to its declared state.

J. THROTTLED UPDATE SCHEDULER (axm apply --idle)
--------------------------------------------------------------------------------
* Description: A background update scheduler that protects system performance.
* Mechanics:
  - Throttled Execution: Limits the CPU usage and disk I/O of background updates
    using Linux control groups (cgroups) and low-priority I/O scheduling (ionice).
  - Automatically suspends or throttles operations if active high-performance
    processes (such as games or video compilation) are detected.

K. UNIVERSAL PERMISSION CONTROL (axiom-locker)
--------------------------------------------------------------------------------
* Description: A single permission manager to control application access across
  all sandboxing engines.
* Mechanics:
  - axiom-locker: Manages access permissions (camera, microphone, file paths)
    across all packaging architectures using both a TUI and CLI interface.
  - Native Flatpak Behavior: Automatically configures Flatpak permissions upon installation
    to behave as if they were native apps (granting standard path bindings and hardware access).

L. TUI WELCOME WIZARD (axiom-welcome)
--------------------------------------------------------------------------------
* Description: First-boot terminal configuration helper for system provisioning.
* Mechanics:
  - axiom-welcome: Launches an interactive TUI panel right after installation to guide the user.
  - Facilitates core driver configurations, system preferences, and the initial selection/installation
    of containerized Distropaks.

M. TUI APPSTORE CONSOLE (axiom-appstore)
--------------------------------------------------------------------------------
* Description: A keyboard-navigable terminal application storefront.
* Mechanics:
  - axiom-appstore: A lightweight, visual package browser (inspired by Pamac and Shelly).
  - Allows users to search, review descriptions, and queue applications for install
    without utilizing complex shell syntax.

N. UNIFIED DESKTOP STYLING ENGINE (axiom-theme)
--------------------------------------------------------------------------------
* Description: Translates styling configurations system-wide across all desktop environments.
* Mechanics:
  - axiom-theme: Listens for theme, wallpaper, cursor, and icon modifications.
  - Automatically translates and writes the styling properties to GTK, Qt, and XFCE desktop
    registries simultaneously, ensuring visual consistency when switching environments.

O. EMERGENCY RESCUE BOOT CONSOLE (axiom-rescue)
--------------------------------------------------------------------------------
* Description: Minimal initramfs rescue environment for single-key system recovery.
* Mechanics:
  - axiom-rescue: Pre-loader initramfs prompt triggered if system boot fails or selected from GRUB.
  - Prompts with a 1-key restore trigger ('R') that automatically rolls back the Ext4 OverlayFS / rsync
    state to the last working snapshot in under 5 seconds.

P. SMART DEPENDENCY RESOLVER (axm resolve)
--------------------------------------------------------------------------------
* Description: Automatic multi-engine resolution utility for package collision errors.
* Mechanics:
  - axm resolve: Triggered when host APT package dependencies conflict during Sid updates.
  - Scans Flatpak and active Distropak containers (Arch, Alpine, Fedora) and suggests 1-click
    alternative sandboxed installation routes.

Q. VISUAL STATE DIFF VIEWER (axm diff)
--------------------------------------------------------------------------------
* Description: Terminal diff inspector comparing live system state against historical snapshots.
* Mechanics:
  - axm diff: Generates a color-coded terminal view highlighting package manifest changes
    and configuration file alterations relative to previous timestamps or transaction IDs.

R. CRYPTOGRAPHIC BINARY AUDITOR (axm audit)
--------------------------------------------------------------------------------
* Description: Cryptographic verification tool auditing system files against GPG signatures.
* Mechanics:
  - axm audit: Scans host binaries and library hashes against upstream distribution GPG signatures
    to detect file corruption, unverified modifications, or rootkit tampering.
