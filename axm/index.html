AXM Engine & Distropaks Cheatsheet

AXM is the high-performance C-compiled declarative package engine powering Axiom GNU/Linux. It handles native host packages and multi-distro containerized Distropaks seamlessly in under 1ms.

[1] Distropak Source Tags

axm pkg-add --source-select

Source Tag Target Distribution / Format Description & Use Case
(1) latest Fedora Rawhide Absolute latest upstream development packages directly from Fedora development branch.
(2) sandbox Flatpak / Snap / AppImage Universal desktop application sandboxes with automatic path permissions via axiom-locker.
(3) stable Debian Stable Rock-solid long-term release packages for mission-critical software.
(4) secure Fedora Linux Enterprise-grade stable packages with SELinux hardened sandboxing.
(5) rolling Arch Linux / CachyOS Bleeding-edge rolling release packages, AUR access, and high-performance kernel builds.
(6) fast Alpine Linux Ultra-lightweight musl-based micro containers for instant execution.
(c) compile Gentoo Linux Source-compiled binaries tuned specifically for your CPU microarchitecture.
(ctrl + ;) !system! Axiom Native Host Native host system installation requiring physical keyguard bypass.

[2] AXM CLI Commands Reference

Command Function & Behavior
sudo axm pkg-add [PACKAGE] Add package with interactive source selector, keyguard (ctrl + ;) & confirmation override (<3 Proceed).
axm pkg-rfsh Refresh all repository databases (resilient against Debian Sid mirror version drifts).
axm pkg-updt Pre-stage and cache all package updates locally without interrupting host tasks.
axm apply Apply pending declarative changes (includes interactive prompts for Gentoo source compilation).
axm apply --idle Run updates in background cgroups with ionice low-priority throttling.
axm run [DISTROPAK] Drop into an ephemeral container shell that auto-purges when you exit.
axm check Dry-run validation to test configuration syntax and package lists safely.
axm try [PACKAGE] Spin up a Shadow VM microVM overlay of your desktop to test sketchy scripts/packages safely.
axm revert Axiom Time-Machine Ext4 OverlayFS / rsync differential rollback without touching $HOME.
axm diff Display color-coded visual delta between your target configuration and live system state.
axm freeze [PACKAGE] Pin specific package version to prevent accidental upgrades during system updates.
axm thaw [PACKAGE] Unpin package version to allow future updates.
axm export --iso Compile your current system package lists and configuration blueprints into a bootable Live ISO.
