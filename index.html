[01]

AXM, A (somewhat) pseudo-declarative package manager

AXM allows you to run commands like sudo axm pkg-add [PACKAGE] to add a package to your system, sudo axm pkg-rm [PACKAGE] to (take a wild guess) remove a package, as well as respective freeze and thaw commands to keep a package at a single version and for it to not be updated, thaw being to update it post-freeze.

[02]

Axiom is what I'd like to call a 'Centrist Distribution'

Axiom gives you options to use install it with an install script (axiom-install), while also giving you options to bootstrap it. Sure, not the MOST purest thing ever (speaking from experience, i used to require myself to exclusively use Gentoo and LFS for a multitude of reasons.), but at least both are an option! Another instance of this 'centrist' idea comes from the Enterprise-level Stable packages vs. Cutting-Edge packages, you don't have to choose, you get both! (currently im writing this at 4pm, im working pretime for my nocturnal brain, there are more instances of the centrist idea)

[03]

The non-destructive, semi-immutable multi-distro support using Axiom's AXM "Distropaks"'

Native integration for running isolated packages from any distribution inside lightweight, high-performance local containers under a unified command. This comes from utilizing Docker, Podman, and Distrobox to utilize any distro of your choosing. You can pull a distropak by running sudo axm-pull [SUPPORTED DISTRO] (Supported distro list SOON)

Axiom is an independent passion project architected and programmed from scratch by a 13-year-old developer. Made by a coder, for coders. Yes, you read that correctly. I am 13 years old, solo-deving Axiom. Sure, things like the roadmap and certain elements required the help of vibecoding, but all ideas, MOST of the code, and all the love is from me.

[05]

Dual-Mode UEFI & Legacy BIOS

Architected to boot on everything. Automatically detects firmware, configures partition tables, and provisions bootloaders across Legacy BIOS (MBR/GPT) and modern UEFI. Pretty self-explainatory. You can only use GRUB. systemd-boot, limine, EFI Stubs, ect are not included and support is not planned for GRUB though old and buggy is better imo. Don't kill me for that.'

[06]

Easy Interactive Terminal Installer using an archinstall-style install script

The installscript makes it so you do not have to bootstrap your system to install Axiom. complex setups. Features a paginated CLI dashboard (axiom-install) with automated single-partition Ext4 structures and automated ESP/Swap provisioning.

[07]

Live LXQt (that looks eh but works) Desktop Wrapper

Start the desktop on your terms. Boots to a clean command line, launching a fully themed LXQt environment instantly with axiom-live-desktop. Also I have prepackaged Waterfox for a simple yet good web browser, yes somehow Axiom is meant for all X86_64 (amd64) devices that are either old rust-buckets or super-computers

[08]

Declarative System Blueprints using AXMPorts

Configure once, reproduce anywhere. Simpily make sure that the major version of Axiom your using is the same as the axmport you import and bob's your uncle! (A tool to upgrade, check, and mess with your axmport will be coming some day). With AXMPorts you can all local packages via a single YAML file (/etc/axm/axm.yaml), inspired by NixOS reproducibility. This is meant for packages, nothing else. FOR NOW. I plan to make an axiport for overall system configs.

[09]

Rollbacks (ohshitgoback)

State management at your fingertips. You mess up your system by installing a bad systempackage? axm rollback. Rebuild system packages locally and revert changes instantly using the lightweight axm apply and axm rollback engines.

[10]

AXM Shells using Distropaks

By running axm run-shell [DISTROPAK, LEAVE BLANK FOR NATIVE]. That's about it.

[11]

Non-Destructive Migrator (/home transfers)

Transition to Axiom seamlessly. A smart migration installer that safely shrinks your host OS, preserves your personal home files, and imports them cleanly on first boot. First it seperates your home parition from the main OLD partition, renaming it "TRANSFER", in this partition, copying any files will cause them to be transfered during the axiom-migrate wizard.

[12]

Optimized Stable Debian Base

Stable Debian core, you need super powers to break it, optimized for speed. Deeply debloated of bloatware and tuned with custom kernel optimizations for minimal resource footprint and maximum performance.

Part 1: Commit Phase (CM)

Goal: Making Axiom functional, established, and leveled with regular distributions.

CM1 (Commit 1): The Foundation COMPLETED
  • Establish and integrate all brand assets (logos, icons, and typography).
  • Perform initial debloat of the Debian base.
  • Apply core system optimizations for speed, resource management, and hardware compatibility.
CM2 (Commit 2): The Gateway COMPLETED
  • Create a simple, robust, interactive installation script (modeled after the simplicity of archinstall).
  • Ensure reliable partitioning, bootloader installation, and essential system setup.
CM3 (Commit 3): The Desktop Experience COMPLETED
  • Establish Axiom as a full-fledged desktop distribution.
  • Add automated options for proprietary and open-source Graphics drivers.
  • Integrate a custom selection of Desktop Environments and Window Managers.
  • Create a custom-themed LXQt live environment accessible by typing axiom-live-desktop from the live ISO.
CM4 (Commit 4): Polish & Finalization COMPLETED
  • Deeply theme and polish the Desktop Environments (leaving Window Managers stock/clean to respect power-user preferences).
  • Refine user-facing system tools, finalize operating system branding, and resolve outstanding bugs.

Part 2: AlphaBeta Phase

Goal: Implementing the unique, declarative, and containerized hybrid desktop architecture.

Alpha 1: The Declarative Core (Host MVP) IN PROGRESS
  • Establish /etc/axiom/axiom.yaml to track local packages and flatpaks.
  • Develop lightweight Python engine axm apply to resolve package deltas and keyrings local-only.
  • Restrict raw apt commands directly, routing all package actions securely through the axm wrapper.
  • Implement package version pinning and freezing (axm freeze / axm thaw).
Alpha 2: State Rollbacks & Config Management PLANNED
  • Add dotfiles configuration syncing directly to user home directories.
  • Introduce axm rollback to instantly revert package/config states.
Alpha 3: The Axiom Migrator & Boot Profiling PLANNED
  • Develop non-destructive in-place installer migration tool.
  • Safely shrinks host OS, sets up temp cache, and imports user home data cleanly on first boot.
  • Integrate the boot performance diagnostics console (axm boot-profile).
Alpha 4: Container Bridge & Smart Migration PLANNED
  • Out-of-the-box native integration for Podman & Distrobox.
  • Declarative containers: block in axiom.yaml configurations.
  • Automatic analysis of legacy installations to build declarative dist-packages blueprints.
  • Develop axm run to execute commands in temporary, auto-purged Podman sandboxes.
